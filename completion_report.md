# UI 리팩토링 및 기능 업데이트 보고서

## 개요
요청하신 대로 앱의 네비게이션 구조와 프로젝트 진행 흐름을 전면적으로 재구성했습니다. 이제 **프로젝트 - 홈 - 포트폴리오**의 3단 탭 구조를 가지며, 프로젝트는 **상담 → 견적 → 계약 → 시공 → 완료**의 5단계 라이프사이클을 따릅니다.

## 1. 네비게이션 및 레이아웃 변경

### 하단 탭 바 (Bottom Navigation)
하단 메뉴가 다음 3가지 핵심 섹션으로 정리되었습니다:
- **왼쪽**: **프로젝트** (단계별 프로젝트 목록 관리)
- **가운데**: **홈** (전체 일정 및 캘린더)
- **오른쪽**: **포트폴리오** (완료된 프로젝트 갤러리)

### 홈 탭 (Home Tab)
- **통합 캘린더**: 상담부터 완료까지 모든 프로젝트 일정이 색상별로 표시됩니다.
    - **초록색**: 상담 (Consultation)
    - **보라색**: 견적/협의 (Estimate)
    - **파란색**: 계약/시공 (Contract/Construction)
    - **회색**: 완료 (Completed)
- **빠른 추가 (+) 버튼**: 우측 하단에 **플로팅 버튼(+)*이 추가되어, 언제든 즉시 **새 상담 일정**을 등록할 수 있습니다.

## 2. 프로젝트 리팩토링 (왼쪽 탭)

'프로젝트' 탭은 이제 5단계의 진행 과정을 보여줍니다.

### 1단계: 상담 (Consultation)
- **기능**: 예정된 상담 목록을 확인합니다.
- **작업**: 상담 내역을 확인하고 견적 단계로 넘길 수 있습니다.
- **생성**: 홈 탭의 (+) 버튼을 통해 새로운 상담을 생성하면 이곳에 등록됩니다.

### 2단계: 견적 (Estimate)
- **변경 사항**: 기존 견적 폼 내에 있던 '계약서 작성' 단계가 제거되었습니다.
- **완료 액션**: '견적 완료' 버튼을 누르면 프로젝트가 **계약(Contract)** 탭으로 이동합니다. (상태: 계약 대기)

### 3단계: 계약 (Contract) - **신규**
- **목적**: 계약서 작성 및 체결을 위한 전용 공간입니다.
- **워크플로우**: 
    - 견적이 확정된 프로젝트들이 이곳에 표시됩니다.
    - '계약서 작성'을 눌러 표준도급계약서(전자서명)를 작성합니다.
- **완료 액션**: 서명 후 저장을 완료하면 프로젝트가 **시공(Construction)** 탭으로 이동합니다.

### 4단계: 시공 (Construction)
- **기능**: 현재 공사가 진행 중인 프로젝트를 관리합니다.
- **상세 보기**: 기존의 시공 일지, 사진 관리 등의 기능은 그대로 유지됩니다.

### 5단계: 완료 (Completed)
- **기능**: 공사가 최종 완료된 프로젝트 목록입니다.

## 3. 포트폴리오 탭 (오른쪽 탭) - **신규**
- **목적**: 기존의 '분석' 탭을 대체하여, 회사의 시공 실적을 보여주는 공간입니다.
- **기능**:
    - 완료된 프로젝트들을 갤러리 형태로 모아봅니다.
    - **정렬**: 최신순/오래된순 정렬이 가능합니다.
    - **시각화**: 프로젝트의 대표 이미지(또는 AI 모델링 시안)를 썸네일로 보여줍니다.

## 주요 변경 파일 요약
- `HomeView.tsx`, `BottomNavigation.tsx`: 네비게이션 구조 변경
- `ProjectView.tsx`: 5단계 탭 로직 구현
- `ConsultationView.tsx` (신규), `ConsultationModal.tsx` (신규): 상담 기능
- `ContractView.tsx` (신규), `ContractForm.tsx` (신규): 계약 기능
- `EstimateForm.tsx`: 견적 로직 분리 (계약 단계 제거)
- `PortfolioView.tsx` (신규): 포트폴리오 갤러리
- `MonthGrid.tsx`: 캘린더 상태별 색상 구현

기존의 핵심 기능(시공 일지, 견적 항목 작성, AI 모델링)은 새로운 구조 안에서도 안전하게 유지되도록 작업했습니다.
